# SMT 流程对比分析报告

> 日期: 2026-01-19
> 目的: 对比客户提供的 SMT 流程资料与我们系统现有实现的差异

## 一、流程结构对比

| 阶段 | 客户流程 (smt_flow_user.md) | 我们系统 (03_smt_flows.md) | 差异说明 |
|------|---------------------------|--------------------------|---------|
| **工单入口** | SMT工单下达 | ERP/APS 工单下达 → 工单接收(RECEIVED) → 工单释放(RELEASED) | 我们有更细的状态流转 |
| **批次创建** | 无明确批次概念 | 创建批次 (Run=PREP) | ⚠️ **重大差异**：客户流程没有 Run 概念 |
| **准备流程** | 4步顺序：钢网→锡膏→物料→设备 | 4步顺序：钢网→锡膏→物料→设备 + 就绪检查门禁 | 我们多了 Pass/Fail 判断 |
| **上料防错** | 详细的扫码验证流程（逐行比对站位表） | 简化的扫码验证 | 客户更详细 |
| **首件流程** | SPI→回流焊→AOI→首件检测（具体设备节点） | 创建FAI→试产→记录→判定 | 客户有具体检测设备节点 |
| **批量生产** | 多维度监控（设备/工艺/质量/物料）| TrackIn→数据采集→TrackOut | 客户监控维度更丰富 |
| **完工处理** | 转入 DIP/测试工序 | Run=COMPLETED + OQC 可选 | 我们有 OQC 抽检机制 |
| **异常处理** | 有返修流程 | FAIL→不良记录→处置（返修/报废/隔离）+ MRB评审 | 我们有 MRB 评审 |

## 二、客户准备流程详情

- 贴片程序加载

- 顶针设置说明

## 三、客户上料防错流程详情

1. 调取机器《物料站位表》
2. 扫码验证：扫描物料二维码 + 机器站位二维码
3. 系统比对：扫码结果与《物料站位表》逐行对比
4. 防错判断：物料/站位检查
5. 验证结果：正确→确认上料；错误→报警锁定→异常处理→重试

## 四、客户首件流程详情

1. 首件贴装：生产2-3拼板
2. SPI检测：锡膏厚度/面积
3. 回流焊接：监控温度曲线
4. AOI检查：焊点质量检查
5. 首件检测：无丝印物料核对

## 七、核心差异总结

### 7.1 设计理念差异
- **客户偏重"表单驱动"**：每个检查点对应纸质/Excel表单，强调人工记录
- **我们偏重"状态机驱动"**：Run/Unit/FAI 状态流转，强调系统自动化

### 7.2 高优先级缺口（影响追溯合规）
1. **锡膏生命周期管理**：解冻时间、回温时间、搅拌记录、有效期
2. **钢网/刮刀使用次数追踪**：寿命管理
3. **炉温曲线记录**：工艺参数追溯
4. **换料记录**：物料批次追溯

### 7.3 中优先级缺口（增强监控能力）
1. **设备每日点检**：AOI/SPI 开机点检
2. **环境监控**：冰箱温度记录
3. **首件检查详情**：SPI厚度、AOI焊点质量等具体指标

### 7.4 低优先级缺口（Nice to have）
1. 夹具维护记录
2. 钢网清洗记录
3. 生产异常报告电子化

## 八、建议行动

1. **确认客户需求优先级**：上述哪些检查点是"必须电子化"的？
2. **评估数据采集方式**：是系统自动采集（设备对接）还是人工录入？
3. **考虑分阶段实施**：
   - Phase 1：锡膏/钢网生命周期管理（核心追溯）
   - Phase 2：炉温曲线 + 换料记录（工艺追溯）
   - Phase 3：设备点检 + 环境监控（增强能力）
