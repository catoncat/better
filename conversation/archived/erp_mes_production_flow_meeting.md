# ERP订单与MES系统生产流程对齐会议记录

> 日期：2024-12-30
> 参会人：星、陈润山

## 会议主题

本次会议围绕ERP订单与MES系统的生产流程对齐，明确了投料前的状态面板、首件检验、SMT物料核对及产线效率管理等需求。

---

## 核心结论

| 模块 | 决定 |
|------|------|
| 工单面板 | MES新增"投料工单面板"，展示齐料进度（颜色+百分比），替代PM在ERP查询 |
| 数据同步 | MES只读同步金蝶"采购入库单"状态，物料齐料后触发企微通知 |
| 首件流程 | 每个工序（SMT/插件/后焊）独立配置首件检验+QC确认，非整体流程一次性 |
| SMT核对 | 新增扫码核对物料步骤（位置二维码+物料二维码+系统清单三方匹配） |
| 角色权限 | 标准化配置，支持多角色分配，已同步ERP |

## 变动需求点

### 1. 投料工单面板（新增）

- 可视化展示所有待处理订单
- 颜色标识齐料状态（绿/红），百分比显示缺料进度
- 近交货期订单优先展示
- 物料全部入库 → 状态变更为"齐料" → 企微通知PM

### 2. 首件检验逻辑（调整）

- 原理解：整体流程配置一次首件
- 实际需求：**每个工序独立配置**首件检验+QC确认后才能批量生产
- 流程图需调整以反映此逻辑

### 3. SMT扫码核对（新增）

- SMT工序第一步：扫码核对物料
- 核对逻辑：机器位置二维码 + 物料二维码 → 匹配系统工单BOM
- 星后续会提供详细文档说明机器内部逻辑

### 4. 领料状态触发条件（明确）

- 只要物料状态从"未领料"变为任意领料状态 → 即可触发生产通知
- 无需区分"超额"或"全部"

### 5. 设备产能数据采集（待定）

- MES需收集设备产能数据，作为未来自动排产依据
- 具体采集内容待工程部门明确

## 会议要点回顾

### 投料工单状态与齐料面板

由于当前ERP侧MRP运算与采购到货存在延迟，计划在MES系统内增加一个投料工单面板，以替代PM人员在ERP中查询订单。该面板需直观地展示所有待处理订单，通过颜色（如绿/红）及百分比等方式展示物料齐料进度，特别是近交货期的订单。当物料全部入库后，MES系统应能更新工单状态为"齐料"，并通知PM相关人员可开始排产。关键数据源为ERP中的"采购入库单"状态，MES需同步此状态并更新工单。

### 生产排程与作业流程

每道工序（如SMT、插件等）在批量生产前，均需执行"首件"验证，经QC确认无误后方可转入批量生产。MES的生产流程设计灵活，支持针对单个工序配置是否需要"首件"检验及"授权"确认等步骤。SMT机台上的物料装载需要进行扫码核对。MES系统后续需支持在SMT工序中集成此核对逻辑，确保物料装错的概率为零。

### MES系统角色与功能

系统权限角色将实现标准化配置，允许为用户分配多个角色。计划引入TPM管理角色，未来会根据TPM系统提供的设备状态（如是否在维护、报废等）来约束生产。MES将作为数据分析平台，需收集并录入设备产能等数据，作为未来自动排产的依据。
